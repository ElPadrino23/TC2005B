En la práctica usaste pool.query(sql, [valor1, valor2]) con placeholders $1, $2 en lugar de armar el query concatenando strings. Muestra un ejemplo corto de un query inseguro y su versión parametrizada, y explica en 2–3 líneas por qué el segundo bloquea una inyección como ' OR 1=1 --.

Inseguro:

const sql = "SELECT * FROM users WHERE email = '" + email + "'";


Seguro:

const sql = "SELECT * FROM users WHERE email = $1";
pool.query(sql, [email]);


El primer query permite que el usuario inserte algo como ' OR 1=1 -- y altere la consulta.
El segundo trata el input como dato, no como codigo
